# B-3. 시니어 성장용 (설계/운영/확장)

시니어 개발자로 성장하기 위한 고급 지식을 정리합니다.

## 📋 체크리스트

### 아키텍처/모듈 경계
- [ ] **의존성 관리**: 순환 의존성 방지, 의존성 역전 원칙
- [ ] **도메인/뷰/인프라 분리**: Clean Architecture 적용
- [ ] **모듈 경계**: 명확한 모듈 경계 설계

### 디자인 시스템
- [ ] **토큰/variant/컴포넌트 API**: 확장 가능한 컴포넌트 설계
- [ ] **접근성 내장**: 접근성을 기본으로 하는 컴포넌트 설계
- [ ] **문서화**: Storybook 등으로 컴포넌트 문서화

### 성능 엔지니어링
- [ ] **지표 기반 개선**: Core Web Vitals 등 지표 기반 최적화
- [ ] **관측성**: 로깅, 메트릭, 트레이싱 구축
- [ ] **회귀 방지**: 성능 테스트 자동화

### 보안/리스크 관리
- [ ] **위협 모델**: 잠재적 위협 식별 및 대응
- [ ] **공급망**: 의존성 보안 관리
- [ ] **PII/로그 정책**: 개인정보 보호, 로그 관리 정책

### 테스트/CI 전략
- [ ] **테스트 피라미드**: 적절한 테스트 비율 유지
- [ ] **Flaky 관리**: 불안정한 테스트 제거
- [ ] **파이프라인 최적화**: CI 실행 시간 최적화

### 팀 생산성
- [ ] **리뷰 문화**: 건설적인 코드 리뷰 문화 구축
- [ ] **문서**: 지식 공유를 위한 문서화
- [ ] **자동화**: 반복 작업 자동화
- [ ] **기술부채 관리**: 기술부채 식별 및 관리 전략
