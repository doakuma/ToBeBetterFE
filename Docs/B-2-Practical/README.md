# B-2. 실무 적응용 (바로 일하기)

실무에서 바로 적용할 수 있는 핵심 지식을 정리합니다.

## 📋 체크리스트

### 데이터 페칭 운영
- [ ] **로딩·에러·빈 상태**: 일관된 상태 처리 패턴 구현
- [ ] **요청 취소**: AbortController로 불필요한 요청 취소
- [ ] **중복 요청 방지**: 동일 요청 중복 방지 로직
- [ ] **캐시**: TanStack Query 등으로 캐시 관리

### 폼 실무
- [ ] **Validation**: 클라이언트/서버 검증 통합
- [ ] **서버 에러 매핑**: 서버 에러를 사용자 친화적 메시지로 변환
- [ ] **파일 업로드**: 진행률, 취소, 재시도 UX 구현

### 라우팅/권한
- [ ] **보호 라우트**: 인증 필요 라우트 보호
- [ ] **401 처리**: 토큰 만료 시 자동 갱신 또는 로그인 페이지 이동
- [ ] **리다이렉트**: 권한에 따른 페이지 리다이렉트

### UI 품질
- [ ] **모달 스크롤/포커스**: 스크롤 락, Focus trap 구현
- [ ] **접근성 기본**: 시맨틱 HTML, 키보드 내비게이션, ARIA
- [ ] **반응형**: 모바일/태블릿/데스크톱 대응

### 디버깅 루프
- [ ] **재현**: 문제 발생 조건 확인
- [ ] **로그**: 의미 있는 로그 추가
- [ ] **가설**: 원인에 대한 가설 수립
- [ ] **검증**: 가설 바탕으로 수정하고 테스트

### 배포 이슈 대응
- [ ] **환경변수**: dev/stage/prod 환경 분리
- [ ] **캐시**: CDN 캐시 무효화 방법
- [ ] **소스맵**: 프로덕션 디버깅을 위한 소스맵 관리

## 💡 실무 팁

1. **패턴 재사용**: 일관된 패턴을 컴포넌트/훅으로 추상화
2. **에러 처리**: 모든 에러 케이스를 고려한 처리
3. **성능 고려**: 초기 로딩, 리렌더 최적화
4. **접근성**: 기본적인 접근성은 항상 고려
