## 🎯 옵션 B) 트랙별 체크리스트

목표에 따라 필요한 범위를 우선 점검합니다.

### B-1. 취업 면접용 (설명력 중심)

📖 [상세 문서](./B-1-Interview/README.md)

- 웹 동작 원리: 렌더링 파이프라인, 이벤트 루프, CORS
- JavaScript 핵심: 클로저, this, 비동기, 에러 처리
- React 핵심: 상태 설계, useEffect 의존성, 리스트 key, 성능 기본
- CSS 핵심: flex/grid, stacking context, 반응형
- 보안/성능: XSS/CSRF, Core Web Vitals 개념
- 테스트 개념: 단위/통합/컴포넌트/E2E 목적 구분

### B-2. 실무 적응용 (바로 일하기)

📖 [상세 문서](./B-2-Practical/README.md)

- 데이터 페칭 운영: 로딩·에러·빈 상태, 취소, 중복요청 방지, 캐시
- 폼 실무: validation, 서버 에러 매핑, 파일 업로드
- 라우팅/권한: 보호 라우트, 401 처리, 리다이렉트
- UI 품질: 모달 스크롤/포커스, 접근성 기본, 반응형
- 디버깅 루프: 재현 → 로그 → 가설 → 검증
- 배포 이슈 대응: 환경변수, 캐시, 소스맵 기반 추적

### B-3. 시니어 성장용 (설계/운영/확장)

📖 [상세 문서](./B-3-Senior/README.md)

- 아키텍처/모듈 경계: 의존성 관리, 도메인/뷰/인프라 분리
- 디자인 시스템: 토큰/variant/컴포넌트 API, 접근성 내장
- 성능 엔지니어링: 지표 기반 개선, 관측성, 회귀 방지
- 보안/리스크 관리: 위협 모델, 공급망, PII/로그 정책
- 테스트/CI 전략: 테스트 피라미드, flaky 관리, 파이프라인 최적화
- 팀 생산성: 리뷰 문화, 문서, 자동화, 기술부채 관리

## 🚀 사용 방법

### 옵션 A: 3단계 체크 기준

각 항목을 다음 세 단계로 점검합니다:

- **Lv1 (설명 가능)**: 개념, 원리, 트레이드오프를 말/글로 명확히 설명할 수 있습니다.
- **Lv2 (실습 경험)**: 작은 예제/토이/과제로 직접 구현해본 경험이 있습니다.
- **Lv3 (실무 해결 경험)**: 실제 서비스에서 문제를 진단 → 원인 규명 → 수정 → 검증까지 수행해본 경험이 있습니다.

### 옵션 B: 트랙별 체크

목표에 따라 필요한 범위를 우선 점검합니다:

- **취업 면접용**: 설명력 중심으로 핵심 개념 정리
- **실무 적응용**: 바로 일할 수 있는 실무 스킬 중심
- **시니어 성장용**: 설계, 운영, 확장을 고려한 고급 지식
